Creational patterns abstract the instantiation process. They make the object's creation, composition and representation independent from the system.

Such patterns become important when the system evolve to depend more on object composition than class inheritance. As that happens, hard-coding a fixed behaviour becomes less used, compared to defining a smaller set of _fundamental behaviour_ which can compose any number of complex objects.

1. An encapsulation is provided about which concrete classes are used in the system.
2. How each instance is created.

The system is only familiar with the interface which the objects use to communicate.

- flexibility in _what_, _who_, _when_ and _how_ it gets created.
- sometimes creational patterns are competitors, but other times they complete each other and can be used together.
