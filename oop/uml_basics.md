Unified Modelling Language
--------------------------

**Class notation**
	1) Name
	2) Attributes
		- prefixes to indicate visibility:
			- '+' = `public`
			- '#' = `protected`
			- '-' = `private`
	3) Operations
		- prefixes to indicate visibility: the same as for the attributes
	4) Addtitional components
		- responsibilities
		- dependencies
		- etc.

**Interface notation**
	Displayed as circle with the name written under it.

**Collaboration notation**
	Displayed as a dotted ellipse with the name written inside.

**Dependency notation**
	Displayed as a dotted arrow, unfilled at the end and withour borders.
	Its directed towards the independent object.

**Inheritance notation**
	Displayed as an arrow unfilled at the end but with borders.
